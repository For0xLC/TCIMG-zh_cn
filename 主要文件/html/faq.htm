在为Total Commander编写了大量的vbs自动化脚本后，我产生了一个想法：创建一个包含大部分功能的实用程序。从一开始的只有十几个命令，到如今"TCIMG"已经包含数百个命令和参数，可以解决各种问题。此外，TCIMG还可以被其它应用程序调用(如AkelPad等)。
这么做有许多好处：
- 无需为每个具体问题编写新的脚本 - 自带若干命令或参数。

- 代码简洁，没有多余的库、函数和变量，不使用 Include，所有 UDF 函数都针对特定需求重写。所有的大文本都储存在单独的文件中，只在所需的命令中连接。
- 避免修改某个函数时影响到其他命令和参数的稳定性。
- 一行命令行中，可以执行不同的方法与命令，充分体现TCIMG的灵活性和功能。
- 许多命令能够创建/保存/转换初始或最终的数据列表，并传递到其它命令，以解决复杂的任务。
- 所有需要翻译成其他语言的文本都存放在独立的“TCIMG_*.lng”文件中
- 本人作为使用者，喜欢实验，所以大多数设置都存放在独立的文件中，所有人都可以自定义不同的功能
- 节省大量的时间和存储空间...

  1. TCIMG 的用途

TCIMG专为TC、其他应用程序和操作系统的自动化任务而设计。每个电脑用户总有面临重复而单调的任务(如文件操作、批量重命名、创建文件夹、获取信息...)的时候，这些任务通常需要花费几个小时甚至几天的时间。TC简化了这些任务，而"TCIMG"则进一步扩展TC的功能，能够通过一次点击完成大多数任务。

  2. 如何使用 TCIMG？

1) 将下载的TCIMG压缩包解压到任意文件夹中，建议解压到Total Commander的运行目录中，这样就可以使用Total Commander的内部环境变量(详见 附录13 ) 。这样，将TC移动到USB设备或另一台设备上时，所有的按钮依旧能够正常使用。
此外，可以将所有文件复制到Total Commander的根目录下(与TOTALCMD.exe在同一目录)。这样能够使用 %COMMANDER_PATH%\TCIMG.exe作为 TCIMG 的路径。
2) 在工具栏上创建一个按钮并添加所需功能。 以下内容 将介绍如何操作。

  6. 调用 TCIMG 可能出现的错误

以下是可能出现的错误类型：
1) 找不到文件：

检查TCIMG路径是否正确（"命令"字段）。
2) 此功能无法在压缩文件中使用：

大多数命令无法在压缩文件中执行。
支持压缩文件的命令必须在命令之前添加参数 %Z。

%Zinfos=1||2 ;; 显示Total Commander面板信息

3) Error:

出现此错误类型有多种可能：
   - 参数不正确：缺少参数、缺少分隔符...
   - 条件错误：指定的应该是文件夹而不是文件、缺少必需的参数...
   - 代码错误。请通过 TCIMG 网站、mail、ICQ...等方式提交错误，最好附上详细说明和错误提示（如 以上截图 所示)
4) NOT FILE:

文件夹中缺少必要文件 文件列表

5) Function Not Realised! em_CommandTCIMG888

与 Total Commander 相关的命令（如 tcprm）中需要调用参数时，都会先在 %COMMANDER_PATH%\UserCmd.ini 中创建一个临时命令 [em_CommandTCIMG888]，再执行该命令。
出现以上错误，可能是由于：
  - 没有足够时间创建和执行命令。
    增加 TCIMG.ini 中 [Setting] 字段下的 PauseTcPrm 值以解决问题。
  - 写入 UserCmd.ini 文件时出错。
    删除 UserCmd.ini 文件“写”的限制属性（如"只读""隐藏"...）或给予该文件“完全访问”的权限。
若依旧报错，请检查 TCIMG 是否为最新版本。

5) 该命令无法正常工作：
在一些版本的 Total Commander 中，buttn 命令可能执行失败。
无法操作应用程序的窗口: 有些命令对部分应用程序的窗口无效（如浏览器、第三方应用程序...）
遇到类似问题，请提交给我， 或许可以解决问题。
如果出现错误，请检查 TCIMG 是否为最新版本。
最新版本的下载地址： TCIMG.

  7. 调用 TCIMG 的命令行结构

命令行结构：
  - 单命令，命令和参数之间用等号连接：

Command=StringParameters

  - 多命令，命令和参数之间用等号连接，不同命令间用空格分隔：

Command1=StringParameters1 Command2=StringParameters2 ... CommandN=StringParametersN

StringParameters(命令参数)中不能含有空格或双引号！
如果参数中需要使用引号，请用两个单引号( ' ) '' 代替。
如果参数中需要使用空格，请用两个波浪号~~代替。

* 在特殊情况下可以使用引号。注意，使用引号时避免与其他命令或参数“交叉”。
以下示例是关于TC参数“%O”的使用，它必须用引号括住，以免路径中携带空格引起错误：

creat=%p||"%O" ;; 在来源面板中创建一个与所选文件同名但不同后缀的文件

* 如果禁用 NTFS 的 8.3 格式名称命名文件或文件夹，则必须添加引号。 否则，Total Commander的命令行参数（如%p、%t、%n ...）将不会传递短路径作为参数，传递的路径中可能会带有空格（例如 C:\Program Files\ 的短路径形式为 C:\PROGRA~1\），因此必须添加引号：

creat="%p"||"%O" ;; 在来源面板中创建一个与所选文件同名但不同后缀的文件


29.8 版中添加了在命令行中使用注释的功能：
注释的语法：

(§注释内容§)

- 注释不影响命令的执行
- 注释可以出现多次，如：

Command=(§注释一§)StringParameters(§注释二§)

在 TCIMG.ini 的 [Setting] 键下修改 Comment 的值，可以启用/禁用命令行注释功能。

  15. 如何保留 TCIMG 的设置并更新？

自3.7版本以后，TCIMG 的用户配置将独立于 TCIMG.ini 与 TCIMG_Russian.lng。
请使用以下方式保存用户配置：
1. 在 TCIMG.exe 所在目录中新建 TCIMGU.ini 文件，并将 TCIMG.ini 中的需要修改的字段和键值复制到 TCIMGU.ini 中。
2. 在 .\Files\Lng\ 目录下新建 TCIMGU.lng 文件，并将 TCIMG_Russian.lng 中需要修改的字段和键值复制到 TCIMGU.lng 中。 详见 Language

注意！文本文件的编码格式必须是 1200 UTF16 LE(Unicode Little Endian)。
可以直接创建 tcimg.ini 的副本，改名为 TCIMGU.ini，再进行修改。

TCIMGU.ini 和 TCIMGU.lng 是用户配置文件。TCIMG 将优先从 TCIMGU.ini 和 TCIMGU.lng 中读取键值。

TCIMG 更新并增加新的字段时，无须将新字段复制到用户配置文件中，TCIMGU.ini 和 TCIMGU.lng 可以只保存部分字段配置。

3. 为了使更新不会影响到 TCIMG.tcg 中的多行命令，请在 TCIMG 文件夹或"...\Files\Tcg\"目录下创建新的 *.tcg 文件(如 TCIMGU.tcg），并在 TCIMGU.ini 的 [Setting] 字段下添加 TCG 配置项（详见 附录31）。

[Setting]
...
TCG=TCIMGU.tcg

若有多个文件，请用“ / ”符号分隔：

[Setting]
...
TCG=TCIMGU1.tcg/TCIMGU2.tcg

若文件位于 TCIMG 文件夹或“.\Files\Tcg\”以外，请使用完整的绝对（或相对）路径，如：

[Setting]
...
TCG=C:\1\css.tcg/%COMMANDER_PATH%\1\css22.tcg


4. 自16.5版本后 TCIMG.ini 中的 [Setting] 字段下新增键 LNG。
可以使用 "LNG= " 指定 *.lng 文件。
指定lng文件时，使用 TCIMG 文件夹下路径、绝对路径或相对路径。
若有多个文件，请用“ / ”符号分隔：

[Setting]
...
LNG=TCIMGL1.lng/TCIMGL2.lng/C:\1\SSS.lng/%COMMANDER_PATH%\1\LLL.lng

  20. 能否提供 TCIMG 所在文件夹的相对路径？

自 5.0 版本后，TCIMG 自带内部路径变量：
%TCIMG_PATH% - TCIMG 文件夹的路径（不带"\"）
%TCIMG_DRIVE% - TCIMG所在驱动器的盘符（如 D: ）
自 6.4 版本后，又添加了以下变量：
%TCIMG_TEMP% - Temp 临时文件夹的路径，位于 TCIMG 目录下。同“%TCIMG_PATH%\Temp”。清理该文件夹请使用命令 tempxx
自 8.3 版本后，又添加了以下变量：
%TCIMG_INI% - TCIMG.ini 的路径
自 8.5 版本后，又添加了以下变量：
%TCIMG_EXE% - TCIMG.exe 的路径
以上变量未在系统中注册，仅在调用 TCIMG 时使用。
更多路径变量详见附录8。

  21. 可以在Total Commander中使用 TCIMG 的路径变量吗？

可以，需要借助 Autorun 插件。
在 autorun.cfg 中添加对应变量名以及路径即可：
示例：

SetEnv /A TCIMG_DRIVE "%COMMANDER_DRIVE%"
SetEnv /A TCIMG_EXE "%COMMANDER_PATH%\TCIMG.exe"
SetEnv /A TCIMG_PATH "%COMMANDER_PATH%"
SetEnv /A TCIMG_TEMP "%COMMANDER_PATH%\Temp"
SetEnv /A TCIMG_INI "%COMMANDER_PATH%\TCIMG.ini"

这样当TC启动后，便可以使用相应的路径变量，如:

TOTALCMD#BAR#DATA
%TCIMG_EXE%

%TCIMG_EXE%
TCIMG

  23. 如何解决“文件名中带有特殊字符，TCIMG 无法识别”？

大多数示例中使用%L参数传递数据，这种情况下，Total Commander 会以 ANSI 编码创建列表。如果文件/文件夹名中带有特殊字符(Ā， ȍ， Ŵ， ǎ ...)，那么这些特殊字符将会丢失，并生成错误的文件列表，导致一些命令无法正常执行。
可以使用以下的 Total Commander参数：%UL、%UF 、%WL、%WF，它们都支持Unicode。

自10.3版本起，TCIMG 中新增"TCIMGGO.exe"(启动器)，允许 TCIMG 带参数启动，参考附录27

  25. 如果文件名太长，TCIMG 出错怎么解决？

大多数示例使用 %L 参数传递对象列表，在这种情况下，Total Commander 会以 ANSI 编码创建列表。如果文件/文件夹名超过259个字符(完整路径)，由于%L和%UL参数的长度有限制(TC10 中是固定的)，Total Commander 将传递截断的路径给"TCIMG"。出现这种情况，建议使用 %WL 参数来获取路径。
P.S. 不建议创建多级文件夹结构和使用长文件名。大多数程序遇到此类情况时都会遇到问题...

  26. 如何解决“操作%AppData%文件夹中的文件后无法复制”？

如 %PUBLIC%、%WINDIR%、%TEMP%、%APPDATA%、%USERPROFILE%、%ALLUSERSPROFILE%、%LOCALAPPDATA% 等与系统有关的特殊文件夹(或系统变量)或其他程序(如 Total Commander 中的 %COMMANDER_PATH%、%COMMANDER_DRIVE%...)，在 TCIMG 命令中，这些相对路径都会转换成绝对路径。
可以使用 envgt 来禁用此功能。

  27. 如何解决一些命令在处理带有感叹号的文件名时失效的问题？

在编写TCIMG之初， !! 被用在许多命令中充当分隔符。随着时间推移，我发现这是个错误的做法。
自17.2版本以后，新的分隔符 // 加入到这些命令中，以解决这个问题。
在一些命令中， ?? 和 | 也被用作分隔符。
P.S. 感叹号“ ! ”出现在文件名中是符合规范的，因此最好采用 // 作为命令分隔符。

  28. 能否在 TCIMG 中执行其他脚本、程序、模板？

可以.为此我们专门设计了许多命令来拓展用户功能：
  • funau、funjs、funvb 命令可以调用 AU3, JS, VBS 脚本，并获取特定函数的返回值。
  • 参数 ?Y 可以传参调用 AU3, JS, VBS 脚本中的函数来转换信息。
  • scrpt 命令可以运行 AU3, A3X, VBS, JS, HTA, PS1, AHK 等脚本。
  • TCIMG.exe 是编译好的 AutoIt3 脚本，内置AutoIt解释器，能够更好地运行 AU3, A3X 脚本（即不需要另外安装 AutoIt3.exe）, 同时也涵盖了AutoIt3的所有功能。
  • cmdbt 命令可以运行命令行命令、脚本、程序并获取信息流。
  • start 命令可以带不同参数和条件运行程序。
  • 在 TCIMG.ini (TCIMGU.ini) 中可以添加模板、程序路径、更改键值。
  • 几乎所有的窗口命令都有能够更改的设置（颜色、字体、大小等）

P.S. 如果您有有趣的示例、脚本、模板、项目等，您可以将它们发送给我.如果能够发挥作用，它们都有机会被添加到TCIMG中。
